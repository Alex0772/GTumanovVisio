Visio stencil for sorting OrgChart

Instrument for sorting Visio OrgChart diagrams

The stencil contains the service shape and several macros.

Shape helps you start the sorter in manual mode, and the macro sorts the finished OrgChart diagram according to the values of one of the selected data fields.

 Buy on Gumroad

Description

Stencil is a custom addition for the regular builder OrgChart diagrams Visio and works only with diagrams built with the help of a regular builder.

The stencil can perform only one function - sorting shapes in the diagram.

How to install

You can download the stencil to any convenient location on the user's machine. The installation process is not required. But for faster access, it is recommended that you place the stencil in the "My Shapes" folder.

Working with the Stencil

To sort the chart, you must do the following:

  • Open the "Sorting.vss" stencil.
  • Drag&Drop the "Form" master to the drawing page.
  • Select the field by which you want to sort the shapes.
  • If the field has the "Date" data type, select the checkbox "Is date".
  • Press "OK" button.

Auto start

If you use OrgCWiz to build a chart, the sorting process can be started automatically immediately after the chart is built.

Below is an example of a VBS script that opens Visio, runs the OrgCWiz wizard to build a chart on the source data in the OrgData_v2.xls file, then starts sorting in the "Prop.Range" field (not the Date type).

The mSort.Resort procedure takes two parameters:

  • field name for sorting
  • "Is date" (True, False).

Dim vApp

Set vApp = WScript.CreateObject("Visio.Application")

Set objAddOn = vApp.Addons.ItemU("OrgCWiz")

On Error Resume Next

Dim ArgStr

ArgStr = "/FILENAME=""C:\Work\ReSort\TestData\OrgData_v2.xls""" &_

" /NAME-FIELD=""Hierarchy_ID"" /MANAGER-FIELD=""parent_ID""" &_

" /CUSTOM-PROPERTY-FIELDS=Title,Telephone,Range,Date" &_

" /PAGES=""1"" 10" &_

" /DISPLAY-FIELDS=""Hierarchy_ID"",Name_2,Title,Range,Date"

objAddOn.Run ("/S-INIT")

objAddOn.Run ("/S-ARGSTR " + ArgStr)

objAddOn.Run ("/S-RUN ")

Set stD = vApp.Documents.OpenEx ("C:\Work\ReSort\ProductVSS\Sorting.vss", 4)

stD.ExecuteLine "mSort.Resort ""Prop.Range"", False"
